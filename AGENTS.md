# 项目概述

目录下存储了台湾歌手张雨生的资料档案，包括专辑、文章、书信、手稿、访谈、综艺、演出、周边等。网站使用 GitBook 构建，托管在 GitHub Pages（zhang-yusheng.github.io）。

# 目录结构

| 目录 | 内容 |
|------|------|
| `albums/` | 个人专辑、纪念专辑、合作专辑 |
| `articles/` | 张雨生撰写的文章 |
| `letters/` | 书信：家书、情书、致歌迷、致同事 |
| `manuscripts/` | 手稿 |
| `talks/` | 访谈记录 |
| `shows/` | 综艺节目 |
| `performances/` | 演出记录 |
| `goods/` | 周边（书籍等） |
| `memorial/` | 雨生去世后的缅怀节目、纪念特辑 |
| `remix/` | 粉丝二次创作的视频合集、混剪 |
| `audios/` | 音频资料 |
| `videos/` | 视频资料 |
| `img/` | 网站公共图片资源 |
| `css/` | 自定义样式 |
| `more/` | 关于我们、贡献指南、免责声明等 |
| `tosort/` | 待整理的杂物（不纳入版本控制） |
| `scripts/` | 辅助脚本（不纳入版本控制） |

# 命名规范

## 目录命名

每个条目使用独立子目录，格式为 `YYYY-MM-DD-拼音缩写`，如 `1997-10-16-ksxf` 对应 1997年10月16日《口是心非》。日期不精确的部分用 `xx` 替代，如 `1995-xx-xx-lonely-taibei`。

## 图片命名

图片使用歌曲或内容的汉语拼音首字母拼接命名。如《如果你要离开我》的图片命名为 `rgnylkw.jpg`。两首歌曲的图片用 `-` 连接。

## Markdown 文件

一个 markdown 文件对应一个条目（专辑、文章、访谈等）。专辑文件通常包括：文案、歌曲详情（词曲作者、歌词、表演者、文案）、致谢、制作团队、扫描照片、参考链接。扫描照片置于参考链接之前，以单页或对页形式展示。

# 构建与部署

- 基于 GitBook（`book.json`），语言为简体中文
- `SUMMARY.md` 定义导航结构，新增内容需同步更新
- 输出为 HTML 网站，同时提供 epub/pdf/mobi 电子书下载

# 提交规范

提交信息使用中文，格式为 `类型: 简述`，类型包括：
- `feat:` 新增内容
- `fix:` 修正错误
- `chore:` 维护性工作

# 维护原则

- 内容文件使用简体中文
- 日期信息是核心元数据，尽量精确
- 保留原始资料的真实性，校对时注明修改之处
